# Inspiration directory

This directory is for the larger project shape: why ZIP-first matters, what it competes with, how to judge implementations, and what adjacent projects should exist.

## Files

- `vision.md` — product philosophy and north star.
- `comparables.md` — related extensions, APIs, libraries, and address infrastructure.
- `alternatives.md` — plausible ways not to build this as an extension.
- `evaluation-rubric.md` — how to score ZIP-first implementations.
- `future-directions.md` — roadmap and generalization paths.
- `sister-projects.md` — adjacent projects this should inspire.

## Core idea

Address forms should ask for the most informative low-friction field early. In the US, that is often ZIP. Globally, it varies by country. This project should become a small, testable pressure campaign toward better address UX, not only a browser-extension hack.
