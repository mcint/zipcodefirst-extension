Annotated ZIP-first UX

A ZIP-first form should make inference visible, editable, and reversible. ZIP is an input to postal defaults, not a complete address and not a jurisdiction oracle.

ZIP/postal code: explicit low-friction input
↓ defaults
City/state/country: inferred postal defaults, always editable
↓ focuses next
Street address: explicit human input, optionally autocomplete-scoped later
Validation layer: optional later step. Do not conflate with ZIP-first defaulting.
